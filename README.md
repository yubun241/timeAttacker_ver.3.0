# Time Attacker

GPS タイムアタック計測 PWA — GitHub Pages デプロイ前提。

## 特徴

- **マップ上で線を描いて計測ライン定義**（スタート / フィニッシュ / セクション）
- **線分交差判定**による高速通過対応の通過検出（点距離判定の弱点を解消）
- **時刻線形補間**でサンプル間に通過した瞬間の時刻を推定
- **セクションスプリット可視化**（ベスト比較 Δ 表示）
- **localStorage 永続化**（コース・ベストラップ）
- **PWA**（Android ホーム画面追加可、オフラインでも UI 起動）

```bash

# Settings → Pages → Source: main branch / root
# https://<user>.github.io/mirage-timeattack/ で公開
```

## Android での使い方

1. Chrome で公開 URL を開く
2. メニュー → 「ホーム画面に追加」
3. 起動するとフルスクリーン PWA として動作
4. 初回起動時に位置情報の許可を求められるので「常に許可」

## 通過検出アルゴリズム（要点）

従来の「点までの距離 < N メートル」方式は、サンプリング間の移動量が判定半径を超えると検出を失敗します。例えば 100 km/h で 1 Hz GPS だと 28 m/sample → 半径 10 m の円は飛び越えます。

本実装は **2D 線分同士の交差判定**:

1. 直前 GPS 座標 P₁ と現在 GPS 座標 P₂ を線分とする
2. 計測ライン A–B との交差を計算
3. 交差したら、線分上のパラメータ t (0..1) から通過時刻を線形補間: `t_cross = T₁ + (T₂ − T₁) × t`

これによりサンプル間に通過しても確実に検出可能。座標は計測ラインの起点を基準とする等距円筒投影で局所メートル化（サーキット規模なら誤差無視可能）。

## 既知の課題 / 設計トレードオフ

- **GPS 精度**: 一般的なスマホ GPS は ±5–10 m。コンマ秒単位の精度には外付け GNSS が望ましい（Bluetooth GPS を Web Bluetooth で取得する拡張も検討余地）
- **サンプリング間隔**: ブラウザの `watchPosition` は端末依存（多くは 1 Hz）。線分交差方式なのでサンプル間に通過しても拾えるが、補間時刻の誤差は速度 × 半サンプル時間
- **進行方向判定なし（v1）**: 同じ線を逆方向通過しても検出する。サーキット周回ではほぼ問題ないが、追加可能（`side` 値の利用）
